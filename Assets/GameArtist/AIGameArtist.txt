BIÊN BẢN YÊU CẦU KỸ THUẬT (TECHNICAL SPECIFICATION) - V2
DỰ ÁN: WORKFLOW AI GAME ARTIST TRÊN NỀN TẢNG ANTIGRAVITY
________________


1. TỔNG QUAN DỰ ÁN
Mục tiêu: Xây dựng một hệ thống Multi-Agent tự động hóa quy trình sáng tạo cho Game Artist. Hệ thống cho phép chuyển đổi từ bản phác thảo thô (Draft) và ảnh tham khảo (Reference) thành các Concept Art hoàn chỉnh dựa trên các Template có sẵn, đảm bảo tính nhất quán về phong cách và tiết kiệm thời gian sản xuất.


Nền tảng triển khai: Google Antigravity (Sử dụng tính năng Multi-agent Workspace & Artifacts). Công cụ tạo ảnh lõi: Nano Banana (tích hợp trực tiếp bên trong hệ sinh thái Antigravity).


________________


2. KIẾN TRÚC HỆ THỐNG (SYSTEM ARCHITECTURE)
Hệ thống được thiết kế theo mô hình Master-Worker với sự điều phối tập trung qua Antigravity Agent Manager.
2.1. Cấu trúc Workspace
* Workspace A (Art Director Agent): Chuyên xử lý ngôn ngữ, logic và cấu trúc prompt.
* Workspace B (Vision & Layout Agent): Chuyên phân tích hình ảnh phác thảo (Draft) và ảnh tham khảo (Reference).
* Image Worker (Nano Banana Engine): Tích hợp công cụ Nano Banana để xử lý quá trình render hình ảnh trực tiếp.
2.2. Luồng dữ liệu (Data Flow)
1. Input: [Draft Image] + [Reference Image] + [Raw Prompt] + [Template ID].
2. Processing: Các Agent phân tích song song và tổng hợp thành cấu hình lệnh (Prompt/JSON) tối ưu nhất cho công cụ Nano Banana.
3. Execution: Image Worker đẩy lệnh và tài nguyên vào engine Nano Banana.
4. Output: [Final Concept Art] xuất hiện dưới dạng Artifact trong Antigravity.


________________


3. ĐẶC TẢ CHI TIẾT CÁC AGENT
Vai trò Agent
	Nhiệm vụ chính
	Yêu cầu Kỹ năng
	Agent 1: Art Director
	Phân tích Prompt thô, kết hợp với Template ID để tạo bộ Prompt kỹ thuật (được tối ưu hóa cú pháp riêng cho Nano Banana).
	Hiểu sâu về Prompt Engineering và định dạng JSON/cấu trúc tham số mà Nano Banana hỗ trợ.
	Agent 2: Vision Analyzer
	Phân tích bố cục từ ảnh Draft và bảng màu/style từ ảnh Reference.
	Sử dụng Vision Model để trích xuất thuộc tính hình ảnh và định dạng thành input cho Nano Banana.
	Agent 3: QA Reviewer
	Kiểm tra ảnh đầu ra từ Nano Banana so với yêu cầu ban đầu. Đề xuất điều chỉnh prompt nếu ảnh bị lỗi hoặc chưa đúng ý.
	Khả năng so sánh độ tương đồng và đánh giá chi tiết chất lượng ảnh.
	

________________


4. QUY TRÌNH THỰC THI CHI TIẾT (STEP-BY-STEP WORKFLOW)
Bước 1: Tiếp nhận đầu vào (Input Ingestion)
Artist cung cấp dữ liệu qua giao diện chat/inbox của Antigravity:


* File đính kèm: sketch.png, reference_style.jpg.
* Văn bản: "Tạo nhân vật chiến binh cyberpunk, tone màu xanh neon, template: Realistic_Game_Asset".
Bước 2: Xử lý đa phương thức (Multi-modal Processing)
* Agent 1 chuyển đổi thông tin thành một bộ prompt chuyên sâu cho Nano Banana, bao gồm các mô tả về ánh sáng, chất liệu, góc camera.
* Agent 2 trích xuất thông tin nét vẽ từ sketch.png và đặc trưng phong cách từ reference_style.jpg để ghép vào luồng xử lý.
Bước 3: Điều phối Kỹ thuật (Technical Orchestration)
Hệ thống tổng hợp các thành phần thành cấu trúc lệnh JSON để thực thi qua Nano Banana:


* image_prompt: Lời nhắc chi tiết từ Agent 1.
* input_image (Base/Layout): File sketch.png để giữ nguyên bố cục phác thảo.
* style_reference: File reference_style.jpg để đồng bộ phong cách.
* template_parameters: Các thông số thiết lập sẵn của template.
Bước 4: Phê duyệt & Trả kết quả (Review & Approval)
* Ảnh sau khi Nano Banana tạo thành công sẽ được đẩy vào Antigravity Inbox dưới dạng một Artifact.
* Artist có thể xem trước, để lại comment hoặc yêu cầu Agent chỉnh sửa (Iterative refinement) ngay trên Artifact đó mà không cần rời khỏi môi trường Antigravity.

5. YÊU CẦU KỸ THUẬT CHO KỸ SƯ (TECHNICAL REQUIREMENTS)
1. Nano Banana Integration: Cấu hình và thiết lập lời gọi (tool call/API) trực tiếp tới engine Nano Banana từ bên trong Antigravity.
2. State Management: Đảm bảo mỗi phiên làm việc (Session) của Artist lưu giữ đúng Context của ảnh Draft và Ref cho các lần tinh chỉnh tiếp theo.
3. Prompt Optimization Library: Xây dựng thư viện các cấu trúc JSON Prompt chuẩn xác mà Nano Banana hỗ trợ tốt nhất cho các Template của dự án.
4. Artifact Handling: Tự động tạo và cập nhật Artifact khi có phiên bản ảnh mới (Ví dụ: V1, V2...).
