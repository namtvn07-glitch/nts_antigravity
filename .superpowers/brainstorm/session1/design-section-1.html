Thiết kế Phần 1: Agent Orchestration

Phân chia trách nhiệm cho các Agent (Art Director, Vision Analyzer, QA) trong hệ thống.

1. Art Director Agent

  • Đầu vào: User Prompt + Template ID
  • Nhiệm vụ chính: Xử lý Text. Load Template Prompt, bổ sung từ vựng chuyên ngành (lighting, textures), cấu trúc lại thành JSON Prompt chuẩn mực.
  • Đầu ra (State): enhanced_prompt (String).

2. Vision Analyzer Agent

  • Đầu vào: Draft Image (Phác thảo) + Reference Image (Tham khảo kiểu dáng/màu)
  • Nhiệm vụ chính: Xử lý ảnh. Suy luận các thông số trọng số (weight) cho ControlNet. Ví dụ: Draft sẽ map với Canny/Lineart, Reference map với IP-Adapter.
  • Đầu ra (State): vision_config (Dictionary parameters).

3. Image Worker (Native Tool)

  • Hành động: Gom enhanced_prompt và vision_config, thực thi API gọi tới Nano Banana. Tải ảnh kết quả về hệ thống dạng Artifact.

4. QA Reviewer Agent

  • Nhiệm vụ: Dùng Vision Model xem ảnh vừa sinh ra có quá khác biệt so với yêu cầu ban đầu (Base Prompt) không?
  • Hành động: Nếu Pass, thông báo cho User. Nếu Fail, liệt kê các lỗi chi tiết trả ngược về cho Art Director / Vision Analyzer để điều chỉnh Prompt/Weight và chạy lại vòng lặp.

Đánh giá của bạn: Việc tách bạch hoàn toàn chức năng Xử lý text (Art Director) và Xử lý ảnh (Vision Analyzer) có hợp lý với định hướng của Studio không? Bạn có đồng ý với thiết kế phần 1 này không?
